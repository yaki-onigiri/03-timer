# Dev Log

## 2026/03/17

目的
    ポートフォリオ用としてタイマーを作っていく過程で、JavaScriptコードの理解を深める。

実装
    今回は、JavaScriptを使ってタイマー機能を実装した。

    最初にHTMLでタイマーの表示部分とボタンを用意し、その後JavaScriptで動きをつける形で進めた。

    実装の流れとしては、

        ・HTMLの要素を getElementById を使って取得

        ・秒数を管理するための変数を用意

        ・setInterval() を使って、1秒ごとに処理を繰り返す仕組みを作成

        ・秒数を「分:秒」に変換して画面に表示

        ・Start / Stop / Reset ボタンにクリックイベントを設定
    という形で、1つずつ動作を確認しながら進めた。

    コードの構造についてはまだ完全に理解できていない部分もあるが、「表示を更新する処理」と「時間を進める処理」が分かれている構成になっていることは確認できた。

発生したバグ
    「Start」ボタンを押してもカウントアップが始まらないバグに遭遇。

原因
    『時間表示を更新する関数』に含まれる「timeDisplay.textContent =」から下のコードで、小文字と大文字の打ち間違い。（string、padstart）

解決方法
    それぞれの文字を大文字に打ち直す。
        ・string → String
        ・padstart → padStart

学んだこと
    ・JavaScriptは大文字・小文字を厳密に区別する言語であり、単純なスペルミスがエラーの原因になることを実感した。

    ・エラー発生時に、ブラウザのコンソール（F12）を確認することで、原因特定が効率的に行なえることを実体験として学んだ。

    ・setInterval() を使うことで、一定時間ごとに処理を繰り返せることを理解したが、
    どのタイミングで止めるか（clearInterval）についてはまだ理解が浅いと感じた。

    ・秒数をそのまま表示するのではなく、計算して「分:秒」に変換している処理について、仕組みはなんとなく理解できるが、自分でゼロから書けるレベルではないと感じた。

    ・コードは一気に理解しようとすると難しいが、1行ずつ動作を確認しながら進めることで動くものが作れることがわかった。

感想
    「S」の大文字・小文字の判別がつきづらいため、何かとコードに「S」が見受けられる場合は、別のHPで行なっている差分比べの機能を多用していこう。

## 2026/03/18

実装
    タイマー機能を「カウントアップ」から「カウントダウン」に変更し、さらに分・秒の入力機能を追加した。

    最初にHTMLで分・秒の入力欄を追加し、その後JavaScriptでカウントダウンの処理を実装した。

    実装の流れ
        ・入力欄（分・秒）を getElementById を使って取得
        
        ・入力された値を Number() で数値に変換

        ・分と秒を合計して「秒」に変換（min * 60 + sec）

        ・countUp を countDown に変更し、1秒ごとに減少する処理に修正

        ・if文を使い、0秒になったらタイマーを停止する処理を追加

        ・Startボタンの処理を修正し、「初回のみ入力値を反映する」用に変更
    
    という形で、前回のコードをベースに機能を拡張していった。

    コードの構造についてはまだ完全に理解できていないが、
    「状態（seconds）を条件によって制御している」ことが今回新しく理解できた。

発生したバグ
    「Start」ボタンを押してもカウントダウンが始まらないバグに遭遇。

原因
    setInterval() で呼び出している関数名が「countDown」になっているのに対し、実際の関数名が countUp のままになっていたため、関数が存在しない状態になっていた。

解決方法
    関数名を修正し、『呼び出し』と『定義』を一致させた。

学んだこと
    ・エラー文「○○ is not defined」は「その関数や変数が存在していない」ことを意味する。

    ・if文は「条件が true のときだけ処理を実行し、 false のときは何も実行しない」構文であることを理解した。

    ・if (seconds === 0) の条件を使うことで、「最初だけ初期化し、それ以降は状態を維持する」という制御ができることを学んだ。

    ・入力された「分・秒」をそのまま扱うのではなく、「秒」に変換して処理することでロジックがシンプルになることを理解した。

    ・Stop後にStartを押したときにリセットされる問題は、「毎回初期化していたこと」が原因であったため、その解決法として『条件分岐』があることを学んだ。

感想
    if文の使い方について、今まではなんとなくわかっていたのが、しっかりと「状態を制御するための文」であることが理解できた点が収穫になった。

    まだコード全体を完全に理解できているわけではないが、「なぜこの処理が必要なのか」を少しずつ考えながら進められていると感じている。

    これからも1つずつだが、JavaScriptコードの理解を深めていきたいと思う。

実装
    ・タイマー終了時にアラーム音を再生する機能を追加
    ・Audioオブジェクトを使用して「alarm.mp3」を読み込み

    実装の流れ
        ①音声ファイル（alarm.mp3）を用意
        ②audioフォルダを作成し、その中にファイルを配置
        
        ③JavaScriptでAudioオブジェクトを作成
            const alarm = new Audio("audio/alarm.mp3");
        
        ④タイマー終了時に再生処理を追加
            alarm.play();
        
        ⑤ブラウザで動作確認

発生したバグ
・アラーム音が鳴らない
・console に「404エラー」が表示される
・URLに直接アクセスすると「このサイトにアクセスできません」と表示される

原因
・audioフォルダ および alarm.mp3 がGitHub上に存在していなかった
・音声ファイルのパス（audio/alarm.mp3）が正しく解決されていなかった
・ローカル環境とGitHub環境でファイル構成が一致していなかった

解決方法
・audioフォルダとalarm.mp3 をGit管理対象に追加
・git add → commit → push を実行しGitHubに反映  
・Live Serverでローカル環境を再起動し再確認  
・ブラウザをリロードして最新状態を反映

学んだこと
・404エラーは「ファイルが存在しない」ことを意味する  
・パス指定はフォルダ構成と完全一致が必要  
・ローカルで動作しても、GitHub上にファイルがなければ動作しない  
・不具合発生時は「コード」だけでなく「環境・構成」も確認する必要がある  
・Consoleエラーの確認が問題解決の起点になる

感想
今回の不具合はコードミスではなく、ファイル管理の問題だった。
実装だけでなく、ファイル構成・GitHubとの整合性も重要であると理解した。
特に、エラー内容（404）から原因を特定する流れは今後も頻出するため、再現できるようにしておきたい。
