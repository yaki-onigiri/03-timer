# Dev Log

## 2026/03/17

目的
    ポートフォリオ用としてタイマーを作っていく過程で、JavaScriptコードの理解を深める。

実装
    今回は、JavaScriptを使ってタイマー機能を実装した。

    最初にHTMLでタイマーの表示部分とボタンを用意し、その後JavaScriptで動きをつける形で進めた。

    実装の流れとしては、

        ・HTMLの要素を getElementById を使って取得

        ・秒数を管理するための変数を用意

        ・setInterval() を使って、1秒ごとに処理を繰り返す仕組みを作成

        ・秒数を「分:秒」に変換して画面に表示

        ・Start / Stop / Reset ボタンにクリックイベントを設定
    という形で、1つずつ動作を確認しながら進めた。

    コードの構造についてはまだ完全に理解できていない部分もあるが、「表示を更新する処理」と「時間を進める処理」が分かれている構成になっていることは確認できた。

発生したバグ
    「Start」ボタンを押してもカウントアップが始まらないバグに遭遇。

原因
    『時間表示を更新する関数』に含まれる「timeDisplay.textContent =」から下のコードで、小文字と大文字の打ち間違い。（string、padstart）

解決方法
    それぞれの文字を大文字に打ち直す。
        ・string → String
        ・padstart → padStart

学んだこと
    ・JavaScriptは大文字・小文字を厳密に区別する言語であり、単純なスペルミスがエラーの原因になることを実感した。

    ・エラー発生時に、ブラウザのコンソール（F12）を確認することで、原因特定が効率的に行なえることを実体験として学んだ。

    ・setInterval() を使うことで、一定時間ごとに処理を繰り返せることを理解したが、
    どのタイミングで止めるか（clearInterval）についてはまだ理解が浅いと感じた。

    ・秒数をそのまま表示するのではなく、計算して「分:秒」に変換している処理について、仕組みはなんとなく理解できるが、自分でゼロから書けるレベルではないと感じた。

    ・コードは一気に理解しようとすると難しいが、1行ずつ動作を確認しながら進めることで動くものが作れることがわかった。

感想
    「S」の大文字・小文字の判別がつきづらいため、何かとコードに「S」が見受けられる場合は、別のHPで行なっている差分比べの機能を多用していこう。

## 2026/03/18

実装
    タイマー機能を「カウントアップ」から「カウントダウン」に変更し、さらに分・秒の入力機能を追加した。

    最初にHTMLで分・秒の入力欄を追加し、その後JavaScriptでカウントダウンの処理を実装した。

    実装の流れ

        ・入力された値を Number() で数値に変換

        ・分と秒を合計して「秒」に変換（min * 60 + sec）

        ・countUp を countDown に変更し、1秒ごとに減少する処理に修正

        ・if文を使い、0秒になったらタイマーを停止する処理を追加

        ・Startボタンの処理を修正し、「初回のみ入力値を反映する」用に変更
    
    という形で、前回のコードをベースに機能を拡張していった。

    コードの構造についてはまだ完全に理解できていないが、
    「状態（seconds）を条件によって制御している」ことが今回新しく理解できた。

発生したバグ
    「Start」ボタンを押してもカウントダウンが始まらないバグに遭遇。

原因
    setInterval() で呼び出している関数名が「countDown」になっているのに対し、実際の関数名が countUp のままになっていたため、関数が存在しない状態になっていた。

解決方法
    関数名を修正し、『呼び出し』と『定義』を一致させた。

学んだこと
    ・エラー文「○○ is not defined」は「その関数や変数が存在していない」ことを意味する。

    ・if文は「条件が true のときだけ処理を実行し、 false のときは何も実行しない」構文であることを理解した。

    ・if (seconds === 0) の条件を使うことで、「最初だけ初期化し、それ以降は状態を維持する」という制御ができることを学んだ。

    ・入力された「分・秒」をそのまま扱うのではなく、「秒」に変換して処理することでロジックがシンプルになることを理解した。

    ・Stop後にStartを押したときにリセットされる問題は、「毎回初期化していたこと」が原因であったため、その解決法として『条件分岐』があることを学んだ。

感想
    if文の使い方について、今まではなんとなくわかっていたのが、しっかりと「状態を制御するための文」であることが理解できた点が収穫になった。

    まだコード全体を完全に理解できているわけではないが、「なぜこの処理が必要なのか」を少しずつ考えながら進められていると感じている。

    これからも1つずつだが、JavaScriptコードの理解を深めていきたいと思う。

## 2026/03/21

実装
    タイマーの精度問題を改善するため、カウントダウン処理を「終了時刻ベース」に変更した。

背景・課題
    従来の実装では、setInterval によって1秒ごとに seconds を減算していたが、
        ・タブ非アクティブ時に処理が遅延する
        ・リロード時に残り時間がズレる
        ・setInterval の実行タイミングに誤差がある
    といった問題が発生していた。

対応内容
    ①終了時刻ベースへの変更
        ・endTime を新たに導入
        ・スタート時に「現在時刻 + 残り秒数」で終了時刻を設定
            endTime = Date.now() + totalSeconds * 1000;

        ・カウントダウン時は、毎回現在時刻との差分から残り時間を算出
            const diff = endTime - Date.now();
            const remaining = Math.ceil(diff / 1000);
            seconds = remaining;
    
    ②終了判定の修正
        ・秒ではなくミリ秒ベースで判定するように変更
            if (diff <= 0)

    ③ストップ処理の修正
        ・停止時に残り時間を再計算し、seconds に保持
        ・endTime をリセットすることで再開可能にした
    
発生したバグと対応
    ①カウントダウンが表示されない
        原因：remaining を seconds に代入していなかった

        対応：seconds = remaining;
    
    ②初回1秒が反映されない
        原因：setInterval が1秒後に実行されるため

        対応：countDown(); をStart直後に1回実行
    
    ③表示ズレ（2秒飛び）
        原因：Math.floor による切り捨て

        対応：Math.ceil() に変更
    
    ④00:01 で停止する問題
        原因：秒ベースで終了判定していた

        対応：diff <= 0 に変更
    
    ⑤ alert による表示ズレ
        原因：alert がメインスレッドをブロックするため、表示更新が遅れる

        対応：
            setTimeout(() => {
                console.log("時間終了！");
            }, 0);

学び
    ・setInterval は“正確な時間管理”には向かない
    ・時間処理は「状態更新」ではなく「差分計算」で行なうべき
    ・表示用の値（seconds）と、基準となる時間（endTime）は分けて考える必要がある
    ・バグの多くは「設計の混在」によって発生する（旧ロジック ＋ 新ロジック）

今後の改善
    ・アラーム停止ボタンの追加
    ・UIの視認性向上
    ・alert の代替（モーダル表示 など）

実装
    ・残り時間に応じた表示色の変更機能を追加（60秒以下：オレンジ、10秒以下：赤）
    ・進行状況を可視化するプログレスバーを追加
    ・タイマー終了時にメッセージを表示する機能を追加
    ・Stop → Resume の動作を明確化（ボタン表示の切り替え）

    実装の流れ
        ・updateDisplay() に色変更の条件分岐を追加
        
        ・progressBar 要素を取得し、進行度に応じて width を更新する処理を実装
        
        ・countDown() の終了判定内に message 表示処理を追加
        
        ・Start / Stop ボタンに textContent の切り替え処理を追加
            Start → Pause
            Stop → Resume
        
        ・UI更新処理を updateDisplay() に集約

    今回は既存のタイマー機能に対して「UI改善」と「状態の見える化」を目的として機能追加を行った。

発生したバグ
    ・Stop後にResumeすると、残り時間ではなく初期時間から再スタートしてしまうバグ

原因
    ・Resume時に seconds = totalSeconds が実行され、保持していた残り時間が上書きされていたため

解決方法
    ・seconds の代入処理を条件分岐で制御し、「初回のみ」代入されるように修正
    ・再開時は seconds をそのまま使用するように変更
    ・totalSeconds = seconds としてしまうと進行度の基準が変わるため、この処理は削除した

学んだこと
    ・状態（seconds）をむやみに上書きするとバグの原因になる

    ・「いつ代入するか」を制御することが重要
    
    ・UIは状態に応じて変化させる（色・メッセージなど）
    
    ・進行度は「割合（remaining / total）」で管理することで視覚化できる
    
    ・表示処理を1つの関数（updateDisplay）にまとめることで管理しやすくなる

感想
    今回はUIの見た目を改善する実装だったが、実際には「状態管理」の理解が重要であると感じた。

    特に、Resume時のバグは単純なミスではなく、「状態をどう扱うか」を理解していないと発生する問題だった。

    これまでよりも「処理の流れ」だけでなく、「状態がどう変化しているか」を意識するようになってきたと感じる。

    見た目の変更でも内部のロジック理解が必要であり、フロントエンドはUIとロジックが密接に関係していることを実感した。
