## アプリ概要

分・秒を入力してカウントダウンを行なうシンプルなタイマーアプリです。

タイマーの一時停止・再開や、ページリロード後の状態復元など、実用的な機能を備えています。

---

## 技術的なポイント

本アプリでは、タイマーの精度改善を目的として「終了時刻ベース」のロジックを採用しています。

### ■ 終了時刻ベースのロジック

通常の「1秒ずつ減らす方式」ではなく、

**「現在時刻」と「終了時刻」の差分から残り時間を算出する方式** を採用しています。

これにより、以下の問題を防止しています。

- タブ非アクティブ時の遅延
- リロード時のズレ
- setInterval による誤差の蓄積

### ■ setTimeout による再起ループ制御

従来の `setInterval` ではなく、
**setTimeout を用いた再帰処理（tick関数）** に変更しています。

これにより、

- 処理遅延によるズレの蓄積を防止
- タイマー停止の制御が明確化

### ■ 状態管理（Stateオブジェクト）

タイマーの状態を1つのオブジェクトで管理しています。

これにより、

- 状態の一元管理
- バグの発生防止
- 可読性の向上

を実現しています。

### ■ 入力バリデーション機能の追加

タイマーの入力欄に対して、間違った値を入れたときにエラーを出す機能を追加しました。

- 分は「0～99」までしか入力できない
- 秒は「0～59」までしか入力できない
- 「0分0秒」はスタートできない
- 60分を超える時間は設定できない
- 数字以外が入力された場合はエラーになる

・UIの改善

- 入力中にエラーが表示される
- エラーがあるとStartボタンが押せなくなる
- alertをやめて、画面内にメッセージを表示するようにした

・入力チェックを`validateInput`という関数にまとめた
・リアルタイムでチェックする処理を追加した
・HTML と JavaScript の両方で入力制限をかけている

## Features（主な機能）

- カウントダウンタイマー機能（分・秒入力）
- Start / Stop / Reset 機能
- タイマーの一時停止（Stop）と再開（Resume）
- 時間表示（分・秒形式）
- アラーム通知機能
- ページリロード後も状態を維持・復元（localStorage）
- 時間のズレを防ぐ仕組み（終了時刻ベースロジック）
- 進行状況を視覚的に示すプログレスバー
- 残り時間に応じた表示色の変化（60秒以下でオレンジ、10秒以下で赤）
- setTimeout による安定した時間制御
- リアルタイムバリデーション（不正入力防止）
- タイマー進行状況をプログレスバーで表示

## 工夫した点

・`timer !== null` を利用し、``Startボタンの連続クリックによるタイマー重複起動を防止``しています。

・Stop後に再度Startした場合、途中から再開できるように状態を保持

・`try-catch` による``例外処理``で、localStorageのデータが破損時のアプリ停止を防止

・localStorage により、リロード後も状態を復元

・setTimeout の再帰処理により、setInterval のズレ問題を解消

・終了時刻ベースのロジックにより、時間精度を向上

・stateオブジェクトによる状態の一元管理を導入し、複数のグローバル変数による不整合を防止

・UI描画処理を関数分離（render系）して可読性を向上

・音声再生の競合（play と load の衝突）を回避する実装

・入力バリデーションをリアルタイムで実行し、UXを改善
